# 🧪 Руководство Тестового Режима

## Что это такое?

TEST_MODE позволяет вам **создавать VPN ключи БЕЗ РЕАЛЬНОЙ ОПЛАТЫ** для тестирования всей системы.

## Как включить?

### 1. Откройте `server/.env`

```bash
# Найдите строку:
TEST_MODE=true
```

### 2. Это уже включено!

По умолчанию `TEST_MODE=true` - система готова к работе без оплаты.

## Как использовать?

### 1. Запустите сервер

```bash
# Фронтенд
npm run dev

# Backend (в новом терминале)
cd server
npm run dev
```

### 2. Откройте http://127.0.0.1:5173

### 3. Авторизуйтесь

Создайте тестового пользователя или используйте существующего:
- Email: `testuser@example.com`
- Пароль: `Test123456`

### 4. Перейдите в "Мои ключи"

### 5. Нажмите "Купить" на любом плане

✅ Система создаст фиктивный VPN ключ  
✅ Ключ появится в "Ваши активные ключи"  
✅ Можете копировать ID ключа  

## Что происходит в TEST_MODE?

```
Пользователь нажимает "Купить" 149 ₽
    ↓
Frontend отправляет POST /api/purchases
    ↓
Backend проверяет: TEST_MODE=true?
    ↓
ДА → Генерирует фиктивный UUID (без вызова Remnawave)
ДА → Сохраняет в БД
ДА → Возвращает Success
    ↓
Frontend показывает "✓ Ключ создан!"
    ↓
Пользователь видит ключ в "Мои ключи"
```

## Пример фиктивного UUID

```
57fbe6ad-fae3-df88-018c-ee4e90c35986
```

Это автоматически сгенерированный ID на основе:
- Email пользователя
- Номера ключа
- Timestamp

## Переключение на Реальный Режим

Когда готовы переходить на реальную оплату:

### 1. Отключите TEST_MODE

```env
# server/.env
TEST_MODE=false
```

### 2. Убедитесь что есть реальные credentials

```env
REMNAWAVE_BASE_URL=https://panel.coffemaniavpn.online/
REMNAWAVE_TOKEN=eyJhbGciOiJIUzI1NiIsInR5cCI6...
```

### 3. Добавьте реальный способ оплаты

Пока что при `TEST_MODE=false` система создает ключи БЕЗ ОПЛАТЫ.

Следующий шаг: интегрировать реальный платежный шлюз (YuKasha, Stripe и т.д.)

## Где находится код?

### Frontend компонент
`src/app/components/MyKeys.tsx` - Страница "Мои ключи"

### Backend API
`server/routes/purchases.js` - Эндпоинт создания покупок

### Wrapper для Remnawave
`server/remnawave-wrapper.js` - Генерирует UUID в TEST_MODE

```javascript
// В remnawave-wrapper.js:
if (process.env.TEST_MODE === "true") {
  // Генерируем фиктивный UUID
  const mockUUID = generateMockUUID(email, keyNumber);
  return { success: true, user_uuid: mockUUID, ... };
}
```

## Тестирование различных сценариев

### 1. Создать один ключ
- Нажмите "Купить" один раз
- Ключ должен появиться в списке

### 2. Создать несколько ключей
- Нажмите "Купить" несколько раз на разных планах
- Все ключи должны появиться в списке
- У каждого должен быть уникальный ID

### 3. Проверить дату истечения
- Ключ на 1 месяц → дата через месяц
- Ключ на 3 месяца → дата через 3 месяца
- Ключ на 1 год → дата через год

### 4. Копирование ID
- Нажмите кнопку копирования рядом с ID
- ID должен скопироваться в буфер обмена

2. Backend сервер запущен и работает

4. Пользователь авторизован перед покупкой
